# Notes of getting sandbox working

## The issue:
- `code-executor` : is a skill that can execute code in a E2B sandbox environment
- if sandbox return text in stderr, stdout - all is fine
- if it return a file, i can't get the file content especially binary files
- Why?
    - agent is a llm loop over tools till done
    - tools output results are ingested into llm as is assuming it is utf-8
    - downstream write_tool isnt desinged for dealing with not utf-8 input
    - so it will fail
- a related problem
    - was the return values were indicating filenames not pathnames
    - llm was trying to get smart and check if wrote the file using read_file tool.
    - so even when sandbox started to work partially i.e write files - read_file used to fail as it did not have complete pathnames
    - as a result workflow failed.

## Solution:
- ensure SKILL.md for ``code-executor`` can write to file if binary and know where to write
- the delegator also needed system prompt change to steer it away from checking if it wrote a file unless user is specifcally asking
