## Codebase Patterns
- Keep `session` as a gateway-owned host artifact; do not add it to `@adaptive-agent/core` contracts.
- Preserve the v1.4 boundary: `run` is the execution unit, `rootRunId` identifies a run tree, and delegation remains synthetic `delegate.*` tools plus child runs.
- Prefer Bun + Fastify in `packages/gateway-fastify`, with explicit JSON-friendly config and protocol types.
- In `packages/gateway-fastify`, route TypeScript imports through `src/core.ts` so the package can typecheck against the built core runtime without inheriting unrelated `packages/core/src` errors.
- When `packages/gateway-fastify` needs runtime access to `../../core/dist/index.js`, load it lazily inside `src/core.ts`; eager imports make Vitest execute Bun-specific bundle helpers before tests can substitute their own factories.
- Keep WebSocket protocol routing in a pure helper such as `handleGatewaySocketMessage(...)` so protocol behavior can be tested without depending on Bun/WebSocket client teardown semantics.
- Keep routing deterministic and host-owned; bindings and session pinning decide `agentId`, not model output.
- Evaluate binding specificity in `packages/gateway-fastify` as channel, then tenant, then roles, and break equal-specificity ties by config order so routing stays deterministic and auditable.
- Keep the rolling transcript summary on the gateway session record and derive replay envelopes from `session.transcriptSummary` plus the recent raw transcript window; do not move chat history ownership into the core runtime.
- For `message.send`, build the replay envelope in memory first and only persist transcript records after the chat result is known so failed turns do not corrupt prior session history.
- Gateway stores own sessions, transcripts, session-run links, and cron metadata; runtime stores remain authoritative for execution facts.
- Per session, enforce a single write lane and a single active root run while allowing multiple same-principal observers.
- Reuse one orchestrator for WebSocket traffic and cron-triggered work so chat, run, and approval behavior stay aligned.
